# 🔧 修復編譯錯誤

## ❌ 當前問題

Xcode 顯示 2 個錯誤：
```
Unexpected input file: .../CalculatorTableDetailView.swift.temp
Unexpected input file: .../InsuranceCalculatorRow.swift.temp
```

這是因為 `.temp` 檔案仍然在 Xcode 專案中，但已經被重命名，Xcode 找不到原始檔案。

## ✅ 解決方案

### 方法一：在 Xcode 中移除這些檔案（推薦）

1. **開啟 Xcode**
2. **在左側導航欄找到這兩個檔案**：
   - `CalculatorTableDetailView.swift` (顯示為紅色，表示找不到)
   - `InsuranceCalculatorRow.swift` (顯示為紅色，表示找不到)

3. **選中檔案，按 Delete 鍵**
4. **選擇「Remove Reference」**（不要選 Move to Trash）
   - 這樣只會從專案中移除引用，不會刪除實際檔案

5. **重新編譯**（Command + B）

### 方法二：使用終端機清理（如果方法一不適用）

```bash
# 進入專案目錄
cd "/Users/chehungliu/Desktop/superdesign-template拷貝/InvestmentDashboard"

# 清理編譯緩存
rm -rf ~/Library/Developer/Xcode/DerivedData/InvestmentDashboard-*

# 重新編譯
xcodebuild -project InvestmentDashboard.xcodeproj -scheme InvestmentDashboard -destination 'generic/platform=iOS' clean build
```

## 📝 說明

`.temp` 檔案的用途：

- **CalculatorTableDetailView.swift.temp**
  - 試算表詳情視圖
  - 需要 `InsuranceCalculatorRow` Entity 才能使用
  - Core Data 設定完成後，將其改名為 `.swift` 並加入專案

- **InsuranceCalculatorRow.swift.temp**
  - Entity 擴展代碼
  - 需要 `InsuranceCalculatorRow` Entity 才能使用
  - Core Data 設定完成後，將其改名為 `.swift` 並加入專案

## ⚠️ 重要提醒

**不要刪除這些 .temp 檔案！** 它們是完成功能所必需的，只是暫時停用。

當您完成 Core Data 設定後：

1. 將這兩個檔案改名（移除 `.temp`）
2. 在 Xcode 中重新加入專案
3. 取消相關代碼的註解
4. 功能即可完整啟用

## 🔍 驗證

完成後，執行編譯（Command + B），應該看到：

```
** BUILD SUCCEEDED **
```

如果仍有錯誤，請參考 `README_試算表功能啟用.md` 文檔。
